# Известные проблемы и ограничения

## 1. Не открывается экран исходящего вызова

**Тест-кейс:** 50556

### Фактическое поведение

После нажатия на кнопку **"Исходящий вызов"** приложение не открывает ожидаемый экран звонка в эмуляторе.

### Ожидаемое поведение

Должен открываться экран исходящего вызова с ожидаемыми элементами интерфейса, например статусом звонка и именем удалённого участника.

### Артефакты

- `outgoing_call_not_opened.xml`
- `outgoing_call_not_opened.png`

### Влияние

Из-за этого автотест для кейса `50556` завершается со статусом `FAILED`.

## 2. Системный UI входящего вызова не полностью воспроизводится в эмуляторе

**Тест-кейсы:** 50558, 53609 шаги 3.5-11.2, 53610 шаги 3.5-12

### Фактическое поведение

После запуска сценария входящего вызова эмулятор не показывает ожидаемое всплывающее окно, которое необходимо для продолжения следующих шагов.

### Ожидаемое поведение

Должно появляться всплывающее окно входящего вызова, позволяющее продолжить проверку сценария.

### Влияние

Часть тест-кейсов остаётся в статусе `BLOCKED`, так как требуемое состояние интерфейса не воспроизводится стабильно в текущем окружении.
